编译Android Linux Kernel并生成Flashable Zip

本文最后更新于：2020年10月26日 晚上

适用于 Android 10, ARM 64

Clone 所需的 Repository

Clone kernel repo

git clone https://github.com/ExampleUser/ExampleRepo.git

Clone toolchains

# Clang_Google
git clone https://android.googlesource.com/platform/prebuilts/clang/host/linux-x86
# GCC_Google_ARM32
git clone https://android.googlesource.com/platform/prebuilts/gcc/linux-x86/arm/arm-linux-androideabi-4.9
# GCC_Google_ARM64
git clone https://android.googlesource.com/platform/prebuilts/gcc/linux-x86/aarch64/aarch64-linux-android-4.9

Clone AnyKernel3

git clone https://github.com/osm0sis/AnyKernel3

确认 defconfig

比如说 arm64 的内核的 defconfig 一般在 arch/arm64/configs

编译内核

cd ${Your Kernel Path}

清理编译环境

make clean && make mrproper

准备编译环境

mkdir -p out
export ARCH=arm64
export CLANG_PATH=${Your Clang_Google Path}/${Prefered Clang Version}/bin
export PATH=${CLANG_PATH}:${PATH}
export CLANG_TRIPLE=aarch64-linux-gnu-
export CROSS_COMPILE=${Your GCC_Google_ARM64 Path}/bin/aarch64-linux-android-
export CROSS_COMPILE_ARM32=${Your GCC_Google_ARM32 Path}/bin/arm-linux-androideabi-
export LD_LIBRARY_PATH=~${Your Clang_Google Path}/${Prefered Clang Version}/lib64:$LD_LIBRARY_PATH

设置 defconfig

make CC=clang AR=llvm-ar NM=llvm-nm OBJCOPY=llvm-objcopy STRIP=llvm-strip O=out ${defconfig name}

开始编译

make CC=clang AR=llvm-ar NM=llvm-nm OBJCOPY=llvm-objcopy STRIP=llvm-strip O=out -j${$(nproc --all) / 2}

用 AnyKernel 生成 Flashable Zip

cp ${Your Kernel Path/out/arch/arm64/boot/Image.gz-dtb} ${Your AnyKernel3 Path}
cd ${Your AnyKernel3 Path}
zip -r9 flashable.zip * -x .git README.md *placeholder

需要注意的是 AnyKernel 需要一定的改动才能生成 Flashable Zip，详见官方文档。
